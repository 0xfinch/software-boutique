
Software Boutique Application Index Format
============

File:       applications.json
Revision:   2
Date:       December 2016


### Structure Overview

  {
    "Category" {                                  - Application category.
      "application-id" {                          - Unique string identifier for application, apps sorted A-Z. Hyphens preferred.
        "variable": "data",                       - Variable containing single data.
        "list": ["Item 1", "Item 2"]              - Variable containing a 'list' of data.
        "group": { "variable": "data" }           - Group containing data.
       }
     }
  }

 * Standard JSON rules apply. Watch out for the commas.
 * Use &#8217; instead of ' for any apostrophe characters.


### Variable Index

 Variable                Type        Required?   Description
 ----------------------- ----------  ----------  ---------------------------------------------
 name                    string      Yes         Name of the application as displayed to the user.
 launch-command          string      No          Command to launch the installed program.
 description             list        No          Description to display (if none is available via AppStream).
                                                  * Use usual HTML tags for formatting.
 alternate-to            string      No          Mention the app is similar to another product.
 tags                    list        No          Used for filtering applications by interests / types. E.g. "Partitioning", "Audio Production".
 open-source             boolean     Yes         Proprietary or Open Source?
 url-info                string      Yes         URL to the web page for more information.
 url-android             string      No          URL if there is an associated Android app.
 url-ios                 string      No          URL if there is an associated iOS app.
 arch                    list        Yes         Supported architectures for this app.
 releases                list        Yes         Supported versions of Ubuntu MATE to show this application.
 methods                 group       Yes         See below.


### Methods

These define how the application can be installed, removed or upgraded.

 Group      Variable                Type        Required?   Description
 ---------- ------------------- --- ----------  ----------  ---------------------------------------------
 apt        source                  string      Yes         How is this software installed?
                                                            Valid options:
                                                              - main         ㇕
                                                              - universe      | - Enables them if they
                                                              - restricted    |   are disabled.
                                                              - multiverse   ㇘
                                                              - partner           Canonical Partner Repo.
                                                              - ppa:<author/ppa>  Enables a PPA
                                                              - manual            Manually write a source.list

            list-file            *  string                    The source file to be written, including the extension (.list)
            list-key-url            string                    URL for retrieving the signing key.
            list-key-server         list                      OR use this to retrieve the signing key from a server.
                                                              [<server>, <key>]. E.g. ["keyserver.ubuntu.com", "D2C19886"]
            list-source-data     *  string                    Data to write inside the source file.

            main-package            string                    The package used for checking whether the application is installed.
            install-packages        string                    To be installed.
            remove-packages         string                    To be removed.

            upgrade-packages        string                    To be upgraded.

    CODENAME        =   Current Ubuntu release, e.g. "xenial".
