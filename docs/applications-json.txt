
Software Boutique Application Index Format
============

File:       applications.json
Revision:   2
Date:       December 2016


### Structure Overview

  {
    "Category" {                                  - Application category.
      "application-id" {                          - Unique string identifier for application, also used for icon (if using Boutique metadata)
        "variable": "data",                       - Variable containing single data.
        "list": ["Item 1", "Item 2"]              - Variable containing a 'list' of data.
        "group": { "variable": "data" }           - Group containing data.
       }
     }
  }

 * Standard JSON rules apply. Watch out for the commas.
 * Use &#8217; instead of ' for any apostrophe characters.


### Variable Index

 Variable                   Type        Required?    Description
 -----------------------    ----------  ----------   ---------------------------------------------
 ㇗ name                    string      Yes          Name of the application as displayed to the user.
 |
 ㇗ metadata                group       Yes
 |  |
 |  ㇗ origin               string      Yes          "boutique"  = Use metadata specified in applications.json
 |  |                                                "appstream" = Download data via AppStream from internet.
 |  ㇗ distro               string      ↑ appstream  Package release for AppStream, e.g. "xenial", "xenial-updates".
 |  ㇗ component            string      ↑ appstream  Package component for AppStream, e.g. "main", "universe", "multiverse".
 |  |
 |  ㇗ summary              string      ↑ boutique   A brief one-liner that describes the application.
 |  ㇗ description          list        ↑ boutique   A more in-depth description of the application.
 |  ㇗ tags                 list        ↑ boutique   Categories for this application, e.g. "texteditor", "System", "Office", "Application".
 |
 ㇗ launch-cmd              string      No           Command to launch the installed program.
 ㇗ alternate-to            string      No           Mention the app is similar to another product.
 ㇗ open-source             boolean     Yes          Proprietary or Open Source?
 ㇗ urls                    group       Yes
 |  |
 |  ㇗ info                 string      Yes          URL to the web page for more information.
 |  ㇗ android-app          string      No           URL if there is an associated Android app.
 |  ㇗ ios-app              string      No           URL if there is an associated iOS app.
 |
 ㇗ arch                    list        Yes          Architectures to show for this app.
 ㇗ releases                list        Yes          Versions of Ubuntu MATE to show this application.
 ㇗ methods                 group       Yes          How is this software installed?
 |  |
 |  ㇗ apt                  group
 |  |  |
 |  |  ㇗ source               string      Yes          Valid options:
 |  |  |                                                  - "main"         ㇕
 |  |  |                                                  - "universe"      | - Enables them if they
 |  |  |                                                  - "restricted"    |   are disabled.
 |  |  |                                                  - "multiverse"   ㇘
 |  |  |                                                  - "partner"           Canonical Partner Repo.
 |  |  |                                                  - "ppa:<author/ppa>"  Enables a PPA
 |  |  |                                                  - "manual"            Manually write a source.list
 |  |  ㇗ list-file          * string      ↑ Manual    The source file to be written, including the extension (.list)
 |  |  ㇗ list-key-url         string      ↑ Manual    URL for retrieving the signing key.
 |  |  ㇗ list-key-server      list        ↑ Manual    OR use this to retrieve the signing key from a server.
 |  |  |                                                  [<server>, <key>]. E.g. ["keyserver.ubuntu.com", "D2C19886"]
 |  |  ㇗ list-source-data   * string      ↑ Manual    Data to write inside the source file.
 |  |  |
 |  |  ㇗ main-package         string      Yes         The package used for checking whether the application is installed.
 |  |  ㇗ install-packages     string      Yes         To be installed.
 |  |  ㇗ remove-packages      string      Yes         To be removed.
 |  |  |
 |  |  ㇗ upgrade-packages     string      No (→)     To be upgraded. If specified, this will be treated as an opt-in upgrade.
 |  |  |                                               This will ignore "install-packages" and "remove-packages".
 |  |  ㇗ extras               group       No          Extra packages that the user can install in addition.
 |  |     ㇗ "pkg" = "Name"                            - Item  = Packages, comma delimited (','), e.g. "gimp-plugins,gimp-plugins-extra"
 |  |                                                  - Value = Name of packages. e.g. "Extra plugins for GIMP"
 |  |
 |  ㇗ snap                    group
 |     |
 |     ㇗ name                 string                  Name of the snap.
 |
 ㇗ post-install               list        No
    post-remove                list        No
    |
    ㇗ "run=*"                 string      Yes         Run this command afterwards.
    ㇗ "msg=*"                 string      Yes         Present a dialogue box to the user.


  * Marked with an asterisk indicates they can use these placeholders:

    CODENAME        =   Current Ubuntu release, e.g. "xenial".
    OSVERSION       =   Current Ubuntu version, e.g "16.04".
